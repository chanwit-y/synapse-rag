# Epic 6 — RAG Chat

Ask questions against selected knowledge bases and get grounded, multi-provider
answers. (Home page chat, `chatWithRagFromDb` / `chatWithModelFromDb`.)

---

## US-6.1 — Chat against selected knowledge bases

**As a** Researcher, **I want** to pick one or more RAGs and ask a question,
**so that** the model answers from my documents.

**Acceptance criteria**
- I can select the RAG(s) to search and the chat model to answer with.
- The question is embedded and the top-K similar chunks are retrieved across the
  selected RAGs.
- The answer is generated by the selected chat model via the provider-agnostic
  LLM connector.
- With no RAG selected, the chat still answers as a plain model conversation
  (`chatWithModelFromDb`).

---

## US-6.2 — Broaden recall with multi-query expansion

**As a** Researcher, **I want** the system to try alternative phrasings of my
question, **so that** relevant chunks aren't missed due to wording mismatches.

**Acceptance criteria**
- The query is expanded into alternative phrasings by an LLM (prefers OpenAI
  `gpt-4o-mini`, else the selected model).
- All phrasings are retrieved and the results fused with Reciprocal Rank Fusion
  (RRF).
- Expansion is best-effort: a missing key, provider error, ~8s timeout, or
  garbage output falls back silently to searching the original query alone.
- The original language of the question is preserved.

---

## US-6.3 — Apply an AI-instruction template to the chat

**As a** Researcher, **I want** to choose an instruction template,
**so that** the assistant follows a consistent persona/format.

**Acceptance criteria**
- An active AI-instruction template can be selected for the conversation.
- The selected template's content is used as the system prompt
  (`instructionId` → `getContent`).
- With no template selected, a sensible default behavior applies.

---

## US-6.4 — Embed relevant source images in answers

**As a** Researcher, **I want** answers to include the relevant images from the
source documents, **so that** screenshots/diagrams appear inline in context.

**Acceptance criteria**
- Images from the retrieved chunks' source documents are offered to the model in
  retrieval order (most relevant first), up to a capped count.
- The model is instructed to embed only relevant images inline and to not alter
  or invent image paths.

---

## US-6.5 — Test a chat model directly

**As an** Administrator, **I want** to send a message straight to a model
without RAG, **so that** I can verify a model/key is configured correctly.

**Acceptance criteria**
- A direct chat call returns the model's response for a given input.
- Provider/key errors are surfaced clearly via the `ActionResult` contract.
